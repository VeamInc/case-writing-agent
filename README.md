# case-writing-agent

Harvard Business School スタイルのケーススタディを、複数のサブエージェントで順番に作るためのプロジェクトです。

このリポジトリは Web アプリではありません。`Claude Code` 上でこのフォルダを開き、指示を送って使います。

## できること

- ケースのテーマ候補を探す
- ケースクエスチョンを作る
- ケース本文を書く
- Harvard 基準で評価する
- 授業ディスカッションをシミュレーションする

## 初めて使う人向けの流れ

1. このリポジトリを手元に用意する
2. `Claude Code` でこのフォルダを開く
3. チャットでケース作成の依頼を送る
4. ステップごとに内容を確認し、`yes` か修正依頼を返す

## 使い方

### 0. 最初から最後までまとめて進めたいとき

全工程を通して進めたい場合は、最初の依頼文で進め方を指定できます。

`yes` なしで最後まで進めたい例:

```text
ケース作成を始めたい。テーマ: 〜。途中確認なしで最後まで進めて
```

`yes` ありで確認しながら進めたい例:

```text
ケース作成を始めたい。テーマ: 〜
```

最初のように依頼すると、毎回 `yes` を入れなくても最後まで一貫して進めやすくなります。2つ目のように依頼すると、各ステップごとに内容を確認しながら進められます。

### 1. このフォルダを Claude Code で開く

`case-writing-agent` フォルダを Claude Code で開いてください。

### 2. 最初の依頼を送る

たとえば、次のように入力します。

```text
ケース作成を始めたい。テーマ: SaaS企業の値上げ判断
```

または、特定のステップだけ実行することもできます。

```text
case-theme-finder で AI教育企業のテーマを探して
case-question-designer でクエスチョンを作って
case-writer でケースを書いて
case-evaluator で評価して
class-simulator で授業シミュレーションして
```

## 実行の流れ

このプロジェクトでは、5つのステップが順番に進みます。

1. `case-theme-finder`
2. `case-question-designer`
3. `case-writer`
4. `case-evaluator`
5. `class-simulator`

注意点:

- ステップ1だけは自動で開始されます
- ステップ2以降は、毎回あなたの確認が入ります
- 続けるときは `yes`、修正したいときはそのまま修正内容を書けば大丈夫です

## 作成されるファイル

成果物は、ワークスペース直下の `cases/` フォルダに保存されます。

保存イメージ:

```text
cases/
└── {case-slug}/
    ├── theme.md
    ├── questions.md
    ├── case.md
    ├── evaluation.md
    └── simulation.md
```

## 迷ったときの使い方

- 最初は「ケース作成を始めたい。テーマ: ○○」だけで十分です
- 出てきた内容を読んで、「進めて」または `yes` と返せば次に進みます
- 違和感があれば、その場で「もっと経営寄りにして」「日本企業の事例にして」のように修正依頼を出してください

## 補足

詳しい内部ルールは次のファイルにまとまっています。

- `CLAUDE.md`: 全体の進め方
- `AGENTS.md`: サブエージェントの役割一覧
