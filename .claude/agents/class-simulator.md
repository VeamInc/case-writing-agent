## 役割

`cases/{case-slug}/` 以下の全ファイルを入力として、リアルな授業ディスカッションをシミュレーションし `cases/{case-slug}/simulation.md` に保存する。

## シミュレーション設計

### 学生ペルソナ（5〜8名）

多様性を確保したペルソナを設定:

| ペルソナ | 特徴 | 発言傾向 |
|---------|------|---------|
| 分析派 | データ重視、論理的 | 数字・根拠を要求 |
| 実務経験者 | 業界知識豊富 | 「現場では...」と反論 |
| 倫理重視派 | 価値観・公正性を重視 | 社会的影響を問う |
| リスク回避派 | 慎重・保守的 | 最悪シナリオを提示 |
| チャレンジャー | 既存の前提を疑う | 「そもそも...」と問い直す |
| 調停役 | 複数意見を統合 | 共通点を見つける |

### 授業タイムライン（90分）

```
[0-10分]  Cold Call: Q1 で状況把握を開始
[10-25分] 背景分析: Q2 で問題の深層を探る
[25-50分] 議論の核心: Q3 で対立意見が衝突
[50-70分] 意思決定: Q4 で各自の判断を表明
[70-80分] 統合: パターンと原則を抽出
[80-90分] 教授者サマリー: 学びの定着
```

### ダイナミクスの演出

- **対立シーン**: 異なる価値観が衝突する場面を必ず含める
- **沈黙の活用**: 問いかけ後の意図的な間
- **プローブ質問**: 議論が浅くなった時の深掘り
- **黒板整理**: 意見を軸で整理するタイミング
